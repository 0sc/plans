POST /users
Create a user

Create User

This endpoint is used to create a new user profile.

Supported Formats

json

Errors

422 Invalid parameter sent

Examples

// Does not create user with invalid email
POST /users
{
  "user": {
    "name": "Oscar",
    "email": "454@#(*)",
    "password": "tcpu1asg6zv"
  }
}
422
{
  "users": [
    "Email is invalid"
  ]
}
// Does not create user with empty email
POST /users
{
  "user": {
    "name": "Oscar",
    "email": "",
    "password": "fvbuxyd1sxx"
  }
}
422
{
  "users": [
    "Email can't be blank",
    "Email is invalid"
  ]
}
// Does not create user with short name

  ]
}

PATCH /users
Update User information

Update User Profile

This endpoint is used to update the user profile details.

Authentication token has to be passed as part of the request. It must be passed as HTTP header(TOKEN).

Supported Formats

json

Errors

401 Unauthorized. Token is probably invalid.
422 Invalid parameter sent
400 Empty request params

Examples

// Returns 422 for request with invalid email
PATCH /users
{
  "user": {
    "name": "Milo Kemmer I",
    "email": "4546$%$",
    "password": "4xbe806agqv"
  }
}
422
{
  "users": [
    "Email is invalid"
  ]
}
// Returns 401 for request with invalid token
PATCH /users
{
  "user": {
    "name": "",
    "email": "yeenia@kaulke.name",
    "password": "password"
  }
}
401
// Returns 422 for request with empty params
PATCH /users
{
  "user": {
    "name": "",
    "email": "",
    "password": ""
  }
}
422
{
  "users": [
    "Name can't be blank",
    "Name is too short (minimum is 2 characters)",
    "Email can't be blank",
    "Email is invalid"
  ]
}
// Returns 400 for request without strong params
PATCH /users
{
  "name": "NewName",
  "email": "new@email.com",
  "password": "newpass"
}
400
{
  "user": {
    "name": "Darlene Sporer",
    "email": "yeenia@kaulke.name",
    "created_at": "2015-11-30T01:18:54.475Z",
    "updated_at": "2015-11-30T01:18:54.499Z"
  }
}
// Returns 200 for request with valid params
PATCH /users
{
  "user": {
    "name": "Eladio Schaefer",
    "email": "email@email.com",
    "password": "uysy4ba18iu"
  }
}
200
{
  "user": {
    "name": "Eladio Schaefer",
    "email": "email@email.com",
    "created_at": "2015-11-30T01:18:54.615Z",
    "updated_at": "2015-11-30T01:18:54.742Z"
  }
}
// Returns 422 if request params is empty
PATCH /users
{
  "user": {
    "name": "",
    "email": "",
    "password": ""
  }
}
422
{
  "users": [
    "Name can't be blank",
    "Name is too short (minimum is 2 characters)",
    "Email can't be blank",
    "Email is invalid"
  ]
}

POST /auth/login

Authenticate User

This endpoint is used to authenticate a user account.

Supported Formats

json

Errors

422 Query params could not be processed. User not yet created or invalid params provided.

Examples

// Logs user in with valid params and returns auth_token
POST /auth/login
{
  "user": {
    "email": "yeenia@kaulke.name",
    "password": "pass"
  }
}
200
{
  "auth_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzUxMiJ9.eyJ1c2VyIjoxLCJleHAiOjE0NDg5MzI3MDN9.yr8T4TjRufkORvAsV5_OLcxIb9-3Dl0klldyBn2kVr2jwxelGRMgCKCUbhEx647bgNv_6Kjvv_KOj1tYHvtw_w"
}
// Returns 422 if email is not found
POST /auth/login
{
  "user": {
    "email": "floie@becker.info",
    "password": "pass"
  }
}
422
// Return 422 with invalid params
POST /auth/login
{
  "user": {
    "email": "",
    "password": ""
  }
}

GET /auth/logout

Logout user

This endpoint is used to terminate access to an authenticated account by invalidating the token if it has not expired.

Authentication token has to be passed as part of the request. It must be passed as HTTP header(TOKEN).

Supported Formats

json

Errors

401 Unauthorized. Token is probably invalid.

Examples

GET /auth/logout
401
// Logs out user
GET /auth/logout
200
// Returns 401 if token is invalid
GET /auth/logout
401
// Returns 401 if user is not logged in
GET /auth/logout
401
